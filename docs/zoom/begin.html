DSP 体験ラボ
What DSP Means

DSP は「音を数にして、数のまま加工して、また音に戻す」ことです。

難しそうに見えますが、やっていることをかなり乱暴に言うと、毎サンプルごとに「今の値」と「少し前の値」を見て、次の値を計算しているだけです。ZOOM の pedal はこの計算を非常に高速に繰り返しています。

Gain

いちばん簡単な DSP は音量変更です。各サンプルに 0.5 を掛ければ半分、2.0 を掛ければ 2 倍です。drive の入口にもまずこの考え方があります。

Filter

低音を通して高音を削る、高音だけを残す、特定帯域だけ持ち上げる。これが filter です。EQ、tone、wah、cab の基礎になります。

Time

少し前の音を覚えておいて後で混ぜる。これが delay です。短く使えば chorus や flanger、長く使えば echo、たくさん重ねると reverb に近づきます。

Sampling

最初の壁はサンプリングです。音の波を、一定間隔の点に切り分けます。

Shannon のサンプリング定理の要点は、「元の音に含まれる最高周波数の 2 倍より速くサンプリングすれば、理論上は情報を取り逃しにくい」ということです。音楽用途で 44.1 kHz や 48 kHz が使われるのはこの考え方に基づきます。

点が十分に細かければ、元の波形の性質をかなり保てます。粗すぎると見間違いが起きます。

この時点で重要なのは 3 つだけです

1
sample rate

1 秒あたりに何回測るか。48 kHz なら 1 秒に 48,000 回です。

2
bit depth

1 回の測定値をどれくらい細かく表すか。粗いと量子化誤差が増えます。

3
aliasing

高すぎる周波数成分が、別の低い音として折り返して見えてしまう現象です。distortion や waveshaping で特に問題になります。

必要条件の目安: sample rate > 2 x 扱いたい最高周波数

厳密には帯域制限や再構成の条件がありますが、初学者向けにはこの理解で十分です。

Filter Basics

filter は「どの帯域を通し、どの帯域を減らすか」を決める装置です。

ギター pedal の tone, EQ, wah, cab, de-esser 的な動きは、だいたい filter の仲間です。コードで見ると、今のサンプルと過去サンプルの重み付き和として書かれることが多いです。

最小のイメージ

入力をそのまま通すのではなく、少し前の値も混ぜると、なめらかになったり、逆に特定の帯域が強調されたりします。これがデジタルフィルタの入口です。

y[n] = a x[n] + b x[n-1] + c y[n-1]

x は入力、y は出力、n は現在サンプル。過去の入力や出力を混ぜることで周波数特性が決まります。

初心者向けには、FIR は「入力だけを見る」、IIR は「出力も記憶する」と覚えると十分です。

Low-pass: 高音を落とす High-pass: 低音を落とす Band-pass: 一部だけ通す Shelf / EQ: ある帯域を持ち上げる
Delay And Reverb

delay は「過去の音をあとで返す」。reverb は「たくさんの小さな delay の塊」と考えると理解しやすいです。

外部資料でも、デジタル delay は buffer に音をためて後で読み出すもの、reverb は複数の delay line と減衰で空間反射らしさを作るものとして説明されています。ZOOM の解析メモでも delay/reverb はまさにその構造でした。

Delay は一本の時間線、reverb は多数の時間線のネットワーク、とイメージすると見通しが良くなります。

ギター向けに言い換えると

A
slapback

かなり短い delay。厚みが出る。

B
chorus / flanger

もっと短い delay を LFO で揺らす。だから「ピッチが揺れる感じ」が生まれる。

C
reverb

単発の反射ではなく、壁や床や天井で何度も反射した尾を作る。アルゴリズム的には多段 delay と filter の組み合わせで作ることが多い。

ZOOM の解析メモでは、delay 系は buffer と feedback、reverb 系は FDN や comb/allpass の構成として整理されていました。初学者はまず「たくさんの delay が空間を作る」と覚えれば十分です。
Impulse Response

cab simulation が急に分かりやすくなる鍵は、impulse response です。

impulse response は、「その装置に一瞬だけパチッと音を入れたら、どう返ってくるか」を記録したものです。線形時不変の系なら、その応答だけで振る舞いをかなり記述できます。

キャビネット IR の直感

ギターアンプの speaker cabinet は、周波数ごとに通りやすさが違い、箱鳴りやマイク位置でも音が変わります。その「通り方」を impulse response として測っておき、dry な音に畳み込みすると cab を通したような音になります。

output = input (*) impulse response

(*) は convolution。難しそうですが、実装の感覚としては「過去サンプルに IR の係数を掛けて足し合わせる」です。

cab sim を FIR 的な処理で実現する、という話の直感図です。

From Basics To ZOOM

この理解があると、ZOOM の解析ページで何を見ているかが急に読めるようになります。

解析ページで見えるもの 初心者向けの意味 DSP の中身のイメージ
Dynamics / Comp / Gate 大きすぎる音を抑えたり、弱い音を持ち上げたりする envelope follower + gain control + tone shaping
Filter / EQ / Wah 周波数ごとの通し方を変える IIR / biquad / shelving / resonant filter
Drive / Amp 歪ませる、アンプらしい応答にする gain + nonlinear waveshaping + tone stack + cabinet 的補正
Cabinet スピーカー箱とマイクの癖を再現する impulse response / FIR に近い考え方
Modulation 揺れ、うねり、広がりを加える short delay + LFO + filter
Delay / Reverb 反射、空間、余韻を作る buffer, feedback, multi-delay network
DSP % その effect が pedal の計算資源をどれくらい食うか フィルタ数、delay line 長、nonlinear 処理、補間、内部状態の多さ
coefficients / tables つまみ位置に応じた音の変わり方の定数表 tone や attack, decay, cutoff などを数値で管理
