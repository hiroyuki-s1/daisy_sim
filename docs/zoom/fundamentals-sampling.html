Core Idea

サンプリングは「音を一定時間ごとに測ること」、量子化は「その測定値を有限桁に丸めること」です。

この 2 つは似て見えて別物です。sample rate は時間方向の細かさ、bit depth は振幅方向の細かさを決めます。時間方向が粗いと aliasing、振幅方向が粗いと quantization noise が目立ちます。

時間方向の細かさ

48 kHz は、1 秒を 48,000 個の区切りに分けて値を読むという意味です。時間軸に対して「どれだけ頻繁に読むか」を決めます。

sample period T = 1 / f_s

48 kHz なら 1 サンプルは約 20.8 µs ごとです。

振幅方向の細かさ

16-bit や 24-bit は、サンプル値を何段階で表現するかです。階段が粗いほど小さい違いを丸め込んでしまいます。

16-bit ≒ 65,536 段階 / 24-bit ≒ 16,777,216 段階

実務では内部処理に float を使って headroom を確保することが多いです。

Nyquist

Nyquist の要点は、「扱いたい最高周波数の 2 倍より速く読む」というルールです。

直感で理解する

波が 1 周する間に 2 点より少ない観測しかなければ、その波がどれだけ細かいかを見分けにくくなります。別の低い波として説明できてしまうので、元の情報を区別できません。

f_s > 2B

B は元信号に含まれる最高周波数帯域の上限です。

音楽制作でどう使うか

可聴帯域を 20 kHz 程度までとみなすなら、44.1 kHz や 48 kHz が実用的な基準になります。より高い sample rate は、可聴帯域そのものを増やすというより、filter 設計や nonlinear 処理を楽にする意味が強いです。

「96 kHz の方が何でも必ず音が良い」ではありません。DSP のどこで何をしたいかで利点が変わります。
Aliasing

aliasing は「高すぎる成分が、もっと低い別の成分として折り返して見える」現象です。

なぜ drive で問題になるのか

clipping や waveshaping のような nonlinear 処理は、新しい高調波を大量に作ります。その高調波が Nyquist 周波数を超えると、上に行かずに下に折り返してきます。これが耳に不自然なジャリつきやざらつきとして出ます。

aliasing を抑える典型手段

手段考え方よく使う場面
入力を帯域制限高域を事前に整理して折り返し元を減らす特殊な synth / 極端な distortion

Quantization

量子化は「丸め誤差」ですが、静かな場面や feedback の中で効いてきます。

丸める

連続値を有限段階に丸めるので、真値との差が残ります。これが量子化誤差です。

積み上がる

多段処理や feedback 系では誤差が蓄積しやすく、ノイズ床や限界動作に影響します。

対策する

float 処理、適切な headroom、必要に応じた dithering やスケーリングで扱います。

Practical View

ZOOM の解析を読むときは、まず「この effect は時間方向で難しいのか、振幅方向で難しいのか」を見ます。

時間方向で難しい例

Delay、Reverb、Pitch 系は buffer、補間、複数 delay line が出てきます。sample rate の意味と buffer 長の感覚が重要です。

振幅方向で難しい例

Drive、Amp、Comp は gain staging、nonlinear 処理、envelope、aliasing 対策が重要です。bit depth や内部 float も効きます。
