基礎3

delay, chorus, flanger, reverb は、全部「時間をずらして混ぜる」仲間です。

時間系エフェクトは一見種類が多いですが、核は buffer と feedback と modulation です。このページでは、一本の delay line から空間系までを連続したものとして説明します。

Delay Line

delay の本体は「少し前のサンプルを取り出して、今の音に混ぜる」buffer です。

一番単純な delay

y[n] = dry * x[n] + wet * x[n-D]

D は何サンプル遅らせるかです。48 kHz なら 480 サンプルで 10 ms、4800 サンプルで 100 ms です。

feedback を足す

buffer[n] = x[n] + feedback * buffer[n-D]

これで繰り返しが生まれます。feedback を上げすぎると自己発振に近づきます。

Interpolation

delay time を滑らかに変えるには、整数サンプルだけでは足りません。

なぜ補間が要るか

delay time を 12.3 サンプルのように中途半端に動かしたいとき、buffer は整数番地しか持ちません。そこで周辺サンプルから中間値を作る interpolation が必要です。

音にどう効くか

補間が粗いと modulation 系でざらつきや不要な色付けが出ます。高品質な delay や pitch 系が重いのは、補間や resampling のコストもあるからです。

解析文書で interpolation や rate conversion 相当の処理が見えたら、それは「時間を連続っぽく扱うための工夫」です。
Reverb

reverb は「大量の反射を、破綻しない形で、自然に減衰させる」問題です。

アルゴリズム reverb の考え方

複数 delay line、comb filter、allpass、damping filter を組み合わせて、初期反射と後期残響を分けて作ることが多いです。FDN はその発展系として、複数 line 間でエネルギーを回します。

convolution reverb の考え方

部屋や装置の impulse response を測って、その応答を丸ごと畳み込みます。cab sim の理解とかなり近いです。
