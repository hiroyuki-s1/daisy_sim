Compressor

Comp は「入力を観察して、あとから通す量を決める」effect です。

リポジトリ内の解析メモでは、COMP 系は envelope detect → gain compute → VCA → tone filter → makeup gain という流れに整理されていました。これは初心者にもかなり説明しやすい構造です。

Comp は「大きい音を見つけてすぐ削る」だけではなく、attack と release によって反応速度を整えています。

Comp を分解するとこう見える

1
Envelope detect

波形そのものではなく、音量の包絡を取り出します。ピック直後の立ち上がりや、減衰していく様子を見る工程です。

2
Gain compute

包絡が threshold より大きいならどれくらい下げるか、小さいならそのままか、という gain を決めます。

3
Tone / makeup

圧縮すると音が暗く感じたり小さく聞こえたりするので、最後に tone と makeup gain を加えることが多いです。

Attack = 反応の速さ Release = 戻る速さ Sense = どれくらい敏感か Level = 最終出力

サンプル単位の直感

env[n] = smooth(abs(x[n])) gain[n] = rule(env[n]) y[n] = x[n] * gain[n]

実際には attack / release を分けたり、tone filter や table lookup を入れたりしますが、入り口はこの理解で足ります。

解析メモとどう結びつくか

zoom-ms50gplus.md には、Comp 系で sensitivity table、auto-level table、tone table、attack table があることが書かれていました。つまり pedal は、つまみ位置ごとに係数を計算し直し、その係数で毎サンプル処理を回していると読めます。

Comp のドキュメントで coefficient table が出てきたら、「検出の速さ」「圧縮量」「補正 EQ」のどれに効いていそうかをまず考えると、読む速度が上がります。
Cabinet

Cabinet は「speaker + box + mic の色」を、応答として押し込んだものだと見ると分かりやすいです。

初心者は cabinet をただの EQ と思いがちですが、感覚としてはそれで半分正解、半分不足です。speaker の周波数特性、箱鳴り、mic の位置や種類の色付けがまとめて効くので、IR や FIR 的な見方が有効です。

箱の中で何が起きるかを逐一シミュレーションする代わりに、結果だけを IR として持つ、という見方です。

Cabinet が効くポイント

1
高域の落ち方

guitar speaker はフルレンジではないので、歪みの耳に痛い高域をかなり整理します。

2
中域のキャラクタ

どの帯域が前に出るかで cabinet らしさが変わります。

3
mic の色

on-axis か off-axis か、近いか遠いかでかなり変わるので、IR には mic の印象も含まれます。

4x12 は低域と押し出し感 1x12 はまとまりやすい off-axis は高域が穏やか

コードの感覚で見る cabinet

y[n] = h[0]x[n] + h[1]x[n-1] + h[2]x[n-2] + ...

h[k] が impulse response の係数です。過去の入力に重みを掛けて足すので、FIR フィルタの形で理解できます。

なぜ amp のあとに cab が欲しいのか

amp sim や distortion は高域に強い成分を大量に作ります。そのままだとジャリジャリで不自然でも、cab sim を通すと「speaker を鳴らした後の現実的な音」に寄ります。だから amp と cab はセットで語られやすいです。

cab の処理は、drive の「歪みを作る工程」ではなく、その後段の「鳴り方を整える工程」と考えると役割分担が見えます。
Reading The Docs

解析ページでは、どこを見れば Comp と Cabinet の本質に触れられるか。

見る場所 Comp での読み方 Cabinet での読み方

次の読み進め方

ここまで来ると、解析ページに出てくる coefficient, filter, table, DSP load がただの数字ではなくなります。次は実際に effect ごとのページで、どのパラメータがどの工程に効いているかを追う段階です。

  1. index.html で Comp 系を開き、Sense, Attack, Level と coefficient table の対応を眺める。
  2. Cabinet 系を開き、speaker / mic 的な色付けを FIR / IR の視点で読む。
  3. そのあと Delay や Reverb に戻ると、buffer と network の話がさらに自然に読めます。
