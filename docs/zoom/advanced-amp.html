上級分冊 Amp 完全版

ZOOM の AMP は 1 台ずつキャラクタを変えながら、同じ DSP の骨格を使い回しています。

ここでは 16 種類の AMP モデルをすべて個別に解説します。単に「この amp は Marshall 系です」で終わらせず、なぜそのようなフィルタ構成や歪み段、追加パラメータが必要になるのかまで掘り下げます。DSP を書いたことがない人でも読み進められるように、共通の信号フローから個別機種まで順番に整理しています。

Shared Architecture

まず共通骨格を見ると、なぜ AMP ごとの差分が「追加パラメータ」と「係数テーブル」に集中するのかが見えます。

AMP 共通フロー

input -> pre filter -> gain dependent EQ -> 4x oversampling -> multi-stage clipping -> presence EQ -> tone stack -> level

実機アンプは真空管、トランス、受動回路が連鎖して音を作ります。ZOOM 側ではそれを、段ごとの IIR フィルタ、オーバーサンプリング、複数回のクリッピング、最後の tone stack へ分解して再構成しています。

つまり、モデルごとの差は「骨格そのもの」より「どこを強く近似したいか」に現れます。

なぜ 4x oversampling が要るのか

アンプ系は nonlinear 処理が多く、歪ませた瞬間に高次倍音が大量に出ます。そのまま 48kHz で処理すると、ナイキストを超えた成分が折り返して alias になります。そこで 192kHz 相当まで持ち上げてから歪ませ、最後に落とし直します。

Why This Design

「なぜそのような実装をするのか」の答えは、実アンプの違いを CPU 制約の中でどこまで残すかにあります。

Pre-EQ を先に置く理由

歪みの前に低域を整理しないと、音がすぐ飽和して輪郭が崩れます。実機でも coupling capacitor や入力インピーダンスが低域量を決めるので、DSP 側でも先に HPF / pre-EQ を入れます。

クリップを複数段に分ける理由

真空管段は一発で完全飽和するのではなく、数段にわたって徐々に圧縮されます。そこで hard clip を一回だけ置くより、段間に EQ を挟みながら複数回処理した方が feel が近づきます。

tone stack を後ろへ置く理由

Fender / Marshall / Mesa は、歪み段の前後関係こそ違っても、最終的に聴こえる質感は tone stack の位置で大きく変わります。後段に置くと、同じ clipping でも「どの帯域を残すか」がモデル差として見せやすくなります。

追加パラメータが少数に限られる理由

Deep、Mode、Structure のような固有ノブは、その機種らしさが大きく変わる場所だけを外に出したものです。全部を細かく可変にすると UI も DSP 係数管理も重くなるため、効く場所だけ残しています。

Structure は連続ノブではなく、複数の voicing を飛び飛びに切り替えるパラメータです。フィルタセットをジャンプテーブル的に切り替える実装と相性が良いです。

なぜこう実装するか: Bogner 系の魅力は微妙な倍音の出方なので、単純な gain 一発では足りません。しかし連続補間で全部を滑らかにすると、元のスイッチ的な性格が薄れます。そこで Structure を離散モードとして持ち、パターンごとに別の EQ / clip 組み合わせを選ぶ設計になります。

機種ごとの差が見やすい比較軸

Vintage と Modern の境界

FD B-MAN、MS 45os、MS 1959 は、入力差と tone stack の比重が大きい vintage 側です。DZ DRV、Recti ORG、XtasyBlue は、追加 voicing パラメータで歪みそのものを切り替える modern 側です。

Clean 主体か、歪み主体か

FD DLXR、FD TWNR、HW100、UK 30A は clean や edge-of-breakup の気持ちよさが本体です。MS 800、Recti ORG、DZ DRV は「歪ませたときの輪郭」を維持することが中心課題です。

追加パラメータの意味

Deep、Fat、Color、Mode、Structure は、単なる EQ ノブではなく「その amp らしさが変わる場所」を抜き出したものです。つまり UI の都合ではなく、モデリング上の優先順位です。
