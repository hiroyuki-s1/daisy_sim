上級分冊 Reverb

Reverb は「空間のそれっぽさ」を、少ない計算でどう破綻なく作るかの技術です。

良い reverb は、単に長い尾があればよいわけではありません。初期反射、密度の上がり方、帯域別減衰、左右の拡散、モジュレーションによる静的感の回避など、複数の観点が絡みます。

Architectures

アルゴリズム reverb には、comb + allpass と FDN という代表的な考え方があります。

comb + allpass

初期のデジタル reverb でよく見る構造です。comb で尾の骨格を作り、allpass で密度を増します。直感的で設計しやすい一方、調整が悪いと metallic になりやすいです。

FDN

複数 delay line 間でエネルギーを回す network です。空間らしい拡散を作りやすく、room/hall 系でよく使われます。係数行列や damping 設計が重要です。

What Sounds Natural

自然な reverb を作るには、密度の時間変化と高域の減衰が鍵です。

Early reflections

壁の近さや空間サイズ感に効きます。短い discrete delay 群として扱われることが多いです。

Late tail

密な反射の塊。ここが粗いと金属的になります。

Damping

高域の減衰がないと不自然に硬く聞こえます。feedback path 内 filter が重要です。

ZOOM Reading

ZOOM の reverb を読むなら、delay line 数と table の意味を先に見るのが効率的です。

Convolution vs Algorithmic

convolution reverb は「実在空間の応答」、algorithmic reverb は「制御しやすい設計空間」です。

convolution

リアルな空間再現に強いですが、編集自由度や計算コストの扱い方が変わります。cab sim に近い発想です。

algorithmic

size、decay、modulation、tone を自在に動かしやすく、pedal 実装に向いています。
