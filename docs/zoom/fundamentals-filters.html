Why Filters Matter

フィルタは「波形を直接いじる」よりも、「どの周波数成分をどう扱うか」で考えると分かりやすいです。

ギターの音色は、倍音バランスが変わるだけで大きく変わります。low-pass は高域を削って丸くし、high-pass は低域の濁りを削ります。band-pass や peaking は wah や EQ の核になります。

Low-pass

高域を落として丸くする。analog delay や cab、tone ノブの基本。

High-pass

低域を削って輪郭を出す。muddy さを避ける時に効く。

Band / Peak

狙った帯域だけ持ち上げたり通したりする。wah、presence、resonance に直結する。

Difference Equation

実装はだいたい「今と昔の値を、重み付きで足す」形です。

最小の形

y[n] = b0 x[n] + b1 x[n-1] + b2 x[n-2] - a1 y[n-1] - a2 y[n-2]

この式の b 側が feedforward、a 側が feedback です。係数の組み合わせによって cutoff、Q、shelf 特性などが決まります。

b0, b1, b2, a1, a2 が何を意味するか

つまみを回すと内部で係数が変わります。Tone を暗くすれば high-frequency の通り方が変わり、Resonance を上げれば特定帯域の山が鋭くなります。つまり UI の背後には係数計算があります。

解析ドキュメントで coefficient table が出てきたら、それは「つまみ位置ごとの周波数応答の設計図」です。
FIR vs IIR

FIR は入力だけを見る、IIR は出力も記憶する。これがまず大枠です。

FIR

過去入力の重み付き和です。常に安定で、線形位相を作りやすいのが強みです。cab IR や linear-phase な補正に向いています。

y[n] = Σ h[k] x[n-k]

IIR

過去出力も使うので効率が良く、少ない係数で急峻な応答を作れます。EQ、tone stack、wah、resonant filter に多いです。

y[n] = Σ b[k]x[n-k] - Σ a[k]y[n-k]
項目FIRIIR
安定性基本的に扱いやすい係数次第で不安定になりうる
計算量長い IR では重くなりやすい少ない係数で強い特性を作れる
位相線形位相を作りやすい一般には位相が曲がる
典型用途cab sim, convolution, correctionEQ, tone, wah, envelope smoothing
Frequency Response

同じフィルタでも「波形の見た目」より「周波数応答」で読む方が実務的です。

見るべき軸

cutoff frequency、gain、Q、slope。この 4 つを意識するとほとんどのフィルタ UI を説明できます。

ギターエフェクトでの読み替え

Tone は low-pass / shelf、Presence は high-mid / high shelf、Wah は動く band-pass、Cab は複雑な複合応答として見られます。名前が違っても、実際には周波数応答を変える装置であることが多いです。

Implementation Notes

実装では「良い設計」だけでなく、「壊れない構造」も重要です。

biquad 化

高次フィルタは 2 次セクションに分けると数値的に扱いやすくなります。

係数更新

ノブ操作で係数が急変すると zipper noise が出るので、補間することがあります。

固定小数点 vs 浮動小数点

DSP の世代によって丸め、headroom、スケーリングの作法が変わります。
