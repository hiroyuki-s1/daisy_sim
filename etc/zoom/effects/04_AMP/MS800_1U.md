## アルゴリズム詳細解析

### 実機 Marshall JCM800 2203 の回路構成

```
Guitar → [Hi/Lo Input Jack] → V1B (1st gain stage, Hi入力のみ)
  → [Preamp Volume] → V1A ("Cold Clipper" - 非対称クリッピング)
  → V2A (2nd gain stage) → V2B (Cathode Follower / バッファ)
  → [TMB Tone Stack (パッシブ)] → [Master Volume]
  → V3 (Long Tail Pair Phase Inverter)
  → EL34 Push-Pull Power Amp → Output Transformer
  → [Presence = NFB回路の高域制御]
```

### 各ブロックの詳細

#### [1] LoHi PreEQ — Hi/Lo入力スイッチモデル

実機JCM800のHi/Lo入力ジャックの違いをモデル化。3段の1次IIRフィルタで構成。

各フィルタは `y[n] = a0·x[n] + a1·x[n-1] + b1·y[n-1]` の形式:

| フィルタ | Lo (Bypass) | Hi (Active) | 役割 |
|---|---|---|---|
| PreEQ_0 | [1, 0, 0] | [0.976, -0.367, 0.391] | V1Bの入力インピーダンス特性 |
| PreEQ_1 | [1, 0, 0] | [0.958, +0.218, -0.110] | ステージ間のフィルタ |
| PreEQ_2 | [1, 0, 0] | [0.991, -0.980, 0.971] | カップリングコンデンサ HPF |

PreEQ_2のDCゲイン: (0.991-0.980)/(1-0.971) = 0.38 → **-8.4dB** のDCカット (カップリングコンデンサのモデル)

#### [2] Gain-Dependent Pre-EQ — ゲインに応じた周波数応答変化

実機では、ゲインを上げるとプリアンプ管の動作点が変わり、周波数特性が変化する。
これを**ゲイン設定に連動するパラメトリックEQ**でモデル化:

| Gain | Freq (Hz) | Gain (dB) | Q | Trim (dB) | 音質的意味 |
|---|---|---|---|---|---|
| 0 (Min) | 260 | -34.0 | 0.1 | -83.0 | ほぼ無音 (実機のVol=0相当) |
| 1 | 250 | -33.0 | 0.1 | -8.0 | クリーンの底、低域大幅カット |
| 2 | 300 | -30.0 | 0.1 | -2.5 | クリーン |
| 3 | 300 | -27.5 | 0.1 | -1.5 | エッジブレイクアップ |
| 5 | 320 | -24.0 | 0.1 | -0.8 | クランチ |
| 7 | 400 | -12.5 | 0.2 | -0.3 | ドライブ、中域が前に出る |
| 10 (Max) | 680 | -3.3 | 0.1 | -0.2 | フルゲイン、680Hzに軽いディップ |

**設計意図**: 低ゲインでは広範な低域カット (-34dB@260Hz) で痩せたクリーンを表現。
ゲインを上げるにつれEQ周波数が260→680Hzに上昇し、カット量が-34→-3.3dBに緩和。
これは**管の飽和による帯域圧縮**をシミュレートしている。

#### [3] オーバーサンプリング

5次IIR Anti-Aliasing Filter (Butterworth/Chebyshev型):
```
H(z) = (0.000887 + 0.003756z⁻¹ + 0.006944z⁻² + 0.006944z⁻³ + 0.003756z⁻⁴ + 0.000887z⁻⁵)
       / (1 - 3.612z⁻¹ + 5.728z⁻² - 4.889z⁻³ + 2.232z⁻⁴ - 0.435z⁻⁵)
```
DC Gain ≈ 1.004 (unity)。非線形処理（クリッパー）のエイリアシングを防止。

#### [4] 多段ゲイン + Dynamic Clipper

メイン信号処理部。`Fx_DRV_JCM_800` (8152 bytes, 2038命令) の大部分がこのブロック。

**Coe配列から推定されるゲイン段**:

| Coe位置 | ゲイン値 | dB | 推定対応 |
|---|---|---|---|
| [30] | 7.604 | +17.6 | V1B (1st gain stage) |
| [56] | 27.332 | +28.7 | V1A "Cold Clipper" (最大ゲイン段) |
| [71] | 7.101 | +17.0 | V2A (2nd gain stage) |
| [95] | 4.821 | +13.7 | V2B Cathode Follower |
| [114] | 7.852 | +17.9 | Power Amp gain |
| [126] | 6.310 | +16.0 | Output Transformer |

各ゲイン段の間にDCブロッキングフィルタが配置:
- Coe[20-24]: HPF (DC gain = 0.058, 強い低域カット = カップリングコンデンサ)
- Coe[50-52]: DC blocker (0.999/-0.999/0.999 ≈ 7.6Hz HPF)
- Coe[115-117]: DC blocker (0.998/-0.998/0.996)

**`JCM_800_CLIPPER_Dynamic`** (40 bytes, 10命令) — **dis6xで完全逆アセンブル済み**:

```c
// 逆アセンブルから復元した等価C関数
float clipper_dynamic(float input, float gain, float threshold) {
    float x = input * gain;       // MPYSP: 入力にゲインを乗算
    if (x >  threshold) x =  threshold;  // CMPLTSP + conditional MV: 上限クリップ
    if (x < -threshold) x = -threshold;  // XOR 0x80000000で符号反転 + 下限クリップ
    return x;
}
```

- **対称ハードクリッパー** (正負同じ閾値)
- gain値はCoeテーブルから `LDW *A6[offset]` でロード（段ごとに異なるゲイン）
- threshold値は `B4` レジスタで呼び出し元から渡される
- 「Dynamic」= メインDSPが**各ゲイン段で異なるgain/thresholdを渡して複数回呼び出す**
- Coe[122-125] のエンベロープフォロワ (α=0.001, 1-α=0.999) がthresholdを動的に制御する可能性あり
- 実機V1Aの非対称クリッピングとは異なり、**対称クリッピングで簡略化**されている

#### [5] Presence EQ — NFBプレゼンス回路の近似

実機のプレゼンス制御は「パワーアンプのNFBループ内の周波数選択フィルタ」。
ZOOM実装では2バンドのパラメトリックEQで近似 (OVS領域で処理):

**Band 2 (ブースト系)**:

| PRSNC | Freq (Hz) | Gain (dB) |
|---|---|---|
| 0-2 | 900 | 0 (フラット) |
| 3 | 900 | +0.18 |
| 4-5 | 300 | +0.4〜+0.5 |
| 6-8 | 200 | +0.8〜+2.0 |
| 9 | 550 | +3.8 |
| 10 | 900 | +5.8 |

**Band 3 (カット系)**: Presence 6以上で-0.15〜-0.8dBのカット、Q=0.4

#### [7] TMB Tone Stack

Marshall方式パッシブトーンスタックのデジタル近似。各バンド11段階のゲインテーブル:

| 設定 | Bass | Middle | Treble |
|---|---|---|---|
| 0 (全カット) | 0.385 (-8.3dB) | 0.638 (-3.9dB) | 0.441 (-7.1dB) |
| 5 (センター) | 1.418 (+3.0dB) | 1.750 (+4.9dB) | 1.753 (+4.9dB) |
| 10 (全開) | 1.875 (+5.5dB) | 1.875 (+5.5dB) | 1.875 (+5.5dB) |

**注**: テーブルの偶数インデックスにはフィルタ係数 (一部は整数値がfloatとして誤読される) が格納され、`TBL_TO_VAL` 関数群で補間される。最大ゲイン 1.875 (= 15/8) は全バンド共通。
